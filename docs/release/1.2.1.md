# Lost Cities: Road Fixes 1.2.1

## Requirements

- Minecraft `1.21.1`
- NeoForge `21.1.235` or newer in the 21.1 line
- Lost Cities `1.21-8.3.10` exactly

## What changed

This hotfix prevents unsafe interchange ramp composition from aborting Lost
Cities chunk generation while a player follows a highway. Close ramp pavement
now resolves as one connected surface without transitively deleting a separate,
clear deck. Built-in grade planning also includes full arterial footprints,
shared terminals, and a level SPUI core.

If an overlay still fails the final clearance gate, Road Fixes atomically omits
its ramps and keeps both graded through highways. The guarded path is used by
both building suppression and late rendering, so the exception cannot escape
through either Lost Cities hook. `/lostcitiesroadfixes status` reports how many
times this containment path was used.

The root cause and test evidence are recorded in the
[highway chunk-generation regression](../regressions/2026-07-highway-chunk-generation-fallback.md)
and [ADR-0024](../architecture/adr-0024-fail-safe-interchange-composition.md).

## Installation

1. Back up the world.
2. Replace version 1.2.0 with `lostcitiesroadfixes-1.2.1+mc1.21.1.jar` on the
   server and every connecting client.
3. Keep Lost Cities `1.21-8.3.10` installed.
4. Test in fresh chunks. Existing road blocks and failed/saved chunks are not
   rewritten.
