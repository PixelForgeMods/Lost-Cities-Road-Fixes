# Lost Cities: Road Fixes 1.0.0

## Requirements

- Minecraft `1.21.1`
- NeoForge `21.1.235` or newer in the 21.1 line
- Lost Cities `1.21-8.3.10` exactly

The Lost Cities version is intentionally exact because Road Fixes validates and
targets internal world-generation methods that are not a stable public API.

## Installation

1. Back up the world.
2. Put `lostcitiesroadfixes-1.0.0+mc1.21.1.jar` in the server's `mods` folder and
   every connecting client's `mods` folder.
3. Keep Lost Cities `1.21-8.3.10` installed.
4. Start the game/server and confirm the compatibility message in `latest.log`.
5. Generate new chunks. Existing saved chunks are deliberately not rewritten.

Install before pregeneration. Removing the addon later changes road generation
for newly explored chunks and can produce boundaries between the two generators.

## Operator checks

Run `/lostcitiesroadfixes status` (or `/lcroadfixes status`) as permission level
2+. Use `/lostcitiesroadfixes clear_caches` after diagnosing a datapack/config
change if an explicit cache reset is needed.

## Release files

- `lostcitiesroadfixes-1.0.0+mc1.21.1.jar`: universal client/server mod
- `lostcitiesroadfixes-1.0.0+mc1.21.1-release.zip`: handoff bundle containing the
  JAR, README, MIT license, changelog, and `SHA256SUMS`

Verified SHA-256 values (two independent clean builds produced the same bytes):

- JAR: `170fd00218f8cc88faa305b8a63ae64ad99767fc0a6694b4c11745b0aa40751b`
- bundle: `4869cf2f3338d691d8c13b565892e45a49a0a6226278407db0bec752200b40ac`

Verify the artifact before distribution:

```text
python3 scripts/verify_release_artifact.py \
  build/libs/lostcitiesroadfixes-1.0.0+mc1.21.1.jar \
  --bundle build/distributions/lostcitiesroadfixes-1.0.0+mc1.21.1-release.zip
```

## Exact-pack result

The client's native setup reproducibly generated zero road cells in the reported
gap chunk. The addon generated a continuous level surface through that chunk and
both neighbors. See [the exact-pack report](../validation/2026-07-exact-modpack.md).

The supplied pack also emits unrelated model, recipe, block-entity, and occasional
Create/Ponder networking errors. None of the captured stack traces implicated
Road Fixes; pack maintainers should triage those separately.
